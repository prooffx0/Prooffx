# ProofX · LUDI Verifier  
“We protect truth so humans can survive the age of synthetic lies.”

---

## 1. What is ProofX?

**ProofX is a trust layer for the AI era.**  
우리는 이미지, 문서, 계약서, 메시지, 영상 등 “생성된 모든 것”에 **위조 불가능한 생성 시점 기록 (Immutable Origin Proof)** 을 남기고, 이후에 누구나 그 진위를 검증할 수 있게 합니다.

이 트러스트 레이어의 심장부가 바로 **LUDI Verifier** —  
“이거 진짜야?”라고 물었을 때 답해주는 공개 검증자입니다.

> ProofX = 기록  
> LUDI Verifier = 검증  
> → Together: a defense system for human truth.

## 2. The Problem: “What’s real?”

우리는 지금, 디지털에서 “그게 진짜 있었던 일이야?”라는 질문이 너무 쉽게 나오는 시대에 살고 있습니다.

- 보이스피싱, 계정 사칭, 위조된 송금 요청  
- 조작된 대화 캡처와 존재하지도 않은 약속을 실제 합의처럼 보이게 만든 메시지  
- 한 사람의 말투와 목소리까지 복제해서 “그 사람이 그렇게 말했다”처럼 연출하는 합성 음성과 생성형 AI

결과는 항상 비슷합니다.  
피해자는 바로 돈·신용·관계를 잃고,  
나중에 “그건 조작된 거예요”라고 설명하러 다닐 때쯤이면 이미 거의 무너져 있습니다.

이건 단순히 “사기꾼이 똑똑해졌다”의 문제가 아닙니다.  
진짜 문제는, **피해자가 자기 자신을 방어하고 증명할 방법이 거의 없다는 것**입니다.

핵심은 이것입니다:

> 내가 사실을 말하고 있어도  
> 그걸 바로 증명할 수단이 없으면  
> 시스템은 나를 보호하지 않는다.

이건 인터넷 문화의 불편함 정도가 아니라  
생활, 신용, 안전, 인간관계, 재산 전체에 영향을 줍니다.

지금은 해외 조직이 사람들을 유인·구속하고, 합성 음성/가짜 계약/조작된 로그를 공장식으로 만들어내는 사례까지 공개적으로 드러나고 있습니다. 위조된 신분, 가짜 지시, 조작된 대화 로그가 대량으로 생산·유통되는 상황은, 더 이상 “개인의 부주의” 문제가 아니라 “디지털 사기의 산업화”가 이미 진행 중이라는 신호입니다.

우리는 이제 이렇게 말할 수밖에 없습니다:

**“진실은 저절로 살아남지 않는다.  
진실은 증명 가능한 형태로 남겨야 한다.”**

ProofX는 바로 그 “증명 가능한 형태”를 만들기 위한 프로젝트입니다.  
우리는 “이건 진짜야”라는 감정적 주장에 기대지 않습니다.  
우리는 “이건 실제로 이렇게 발생했고, 이렇게 변했고, 이 주체가 책임자다”를 기술적으로 제시할 수 있는 구조를 설계합니다.

## 3. Founder Note

이 문제는 이론으로만 적은 게 아닙니다.

나 역시 디지털 사기 상황에 직접 노출된 적이 있고, 그 이후의 과정을 겪으면서 한 가지를 아주 명확하게 보게 되었습니다.

피해자는 이미 흔들린 상태에서 시작합니다.  
그런데도 ‘사실관계 증명’의 책임은 피해자 쪽에 올라옵니다.

- “정말 그런 연락을 받았다는 걸 증명할 수 있나요?”  
- “그 송금 요구가 조작된 거라는 근거는요?”  
- “당신이 속았다는 걸 우리가 어떻게 확신하죠?”

반면 공격자는 혼자 움직이지 않습니다.  
해외 조직은 사람들을 속여 데려와 강제로 사기 시스템에 투입시키고, 합성 음성/가짜 계약/조작된 로그를 공장식으로 만들어냅니다. 이건 한 명의 사기꾼이 아니라 시스템 단위의 공격입니다.

그리고 이런 조작은 더 이상 범죄 전용 공간에만 있지 않습니다.  
SNS 릴스/쇼츠/DM 속에서도  
- “실제 촬영본처럼 보이지만 사실은 편집본”,  
- “실제 대화 캡처처럼 보이지만 사실은 합성”,  
- “실제 결제 증빙처럼 보이지만 사실은 생성형 이미지”  
가 자연스럽게 돌아다닙니다.

결국 우리는 화면을 보고 직관적으로 “저건 증거네”라고 믿을 수 없는 상태에 도착했습니다.

여기서 ProofX의 출발점이 생겼습니다.

ProofX는 추상적인 “세상을 더 착하게 만들자” 선언이 아닙니다.  
ProofX는 현실적인 방어권에 대한 제안입니다.

- 내가 실제로 본 것 / 들은 것 / 동의한 것 / 전달한 것  
- 그 ‘원본’과 ‘수정 이력’을 기록하고  
- 제3자가 확인할 수 있는 검증 보고서 형태로 꺼낼 수 있게 만들고  
- “이건 진짜였다”를 즉시 제시할 수 있도록 하는 것

우리는 “사람을 완벽하게 만들겠다”고 약속하지 않습니다.  
우리는 “거짓으로 사람을 무너뜨리는 방식”을 훨씬 어렵게 만드는 인프라를 만들고자 합니다.

ProofX의 목표를 한 줄로 요약하면 다음과 같습니다:

**디지털 환경에서 — 그리고 앞으로 사람 대신 의사결정을 수행할 AI 에이전트까지 포함해서 —  
누구든 자기 현실을 기술적으로 증명할 수 있게 하는 것.**

## 4. The Solution

### 4.1 ProofX — The Trust Layer  
ProofX는 인터넷 위에 얹히는 **신뢰 인프라 계층(Trust Layer)** 입니다.

- 어떤 컨텐츠(이미지, 문서, 음성, 대화 로그 등)가 “처음으로 생성된 순간”의 원본 상태를 캡처합니다.  
- 그 상태를 **불변 원본 스탬프 (Immutable Origin Stamp)** 로 블록체인에 고정합니다.  
- 그 기록은 삭제, 편집, 레트로액티브 수정이 불가능합니다.

한 문장으로 말하면:

> “원본이 언제 어떤 상태로 만들어졌는지”를  
> 거짓말로 다시 쓸 수 없게 만든다.

이건 앞으로의 인터넷/AI 환경에서 최소 생존 장치입니다.

### 4.2 LUDI Verifier — The Public Verifier  
LUDI Verifier는 ProofX 위에서 작동하는 공개 검증자입니다.

누구나 어떤 데이터/파일/발언을 가져와서 물을 수 있습니다:

- “이건 ProofX에 최초로 찍힌 상태와 동일합니까?”  
- “중간에 변조 이력이 있습니까?”  
- “이 말/이미지/기록은 신뢰 가능한 발신자로부터 나왔습니까?”

LUDI는 “신용을 개인이 구걸하지 않도록” 설계되었습니다.  
(= 피해자가 ‘제발 믿어주세요’라고 빌지 않아도 되게)

기술적으로는 “원본 증명 + 무결성 검증 + 출처 신뢰도 평가”를 수행하는 모듈이며,  
비즈니스적으로는 API / SDK / 파트너 연동의 코어입니다.

## 5. Core Concepts

### 5.1 Proof Stamp (진실의 스탬프)  
모든 중요한 생성물에는 “나는 이 상태로 이 시점에 처음 만들어졌다”라는 **Proof Stamp**가 찍힙니다.  
이건 공개 검증 가능한 온체인/원장 기록입니다.

누구나 확인할 수 있고, 누구의 허락도 필요 없습니다.  
(= 중앙기관 믿으라고 강요하지 않음)

### 5.2 Trust Level (신뢰 레벨)  
Proof Stamp가 축적되면, 그 주체(사람 / 회사 / 디바이스 / AI 에이전트)는 **신뢰 레벨**이 올라갑니다.

신뢰 레벨이 높을수록:

- 허위 공격(가짜 스캔들, 조작 루머)에 덜 취약해집니다.  
- 거래/협업/고용/자원 접근에서 더 나은 조건을 받습니다.  
- “내 말은 내가 증명한다”가 가능해집니다.

우리는 ‘좋아요 수’, ‘팔로워 수’ 같은 인기 점수를 만들지 않습니다.  
우리는 **위조 불가능한 진실 기록 기반의 신뢰 점수**를 만듭니다.

→ 이것이 ProofX의 네트워크 효과이자, 향후 경제 모델의 기반입니다.  
신뢰 자체가 자산화됩니다.

## 6. The ProofX Pledge

ProofX는 아래 원칙에 동의하지 않는 파트너와 협력하지 않습니다.

1. **진실은 권리가 아니라 방어막이다.**  
   누구든 자기 진실을 스스로 보호할 수 있어야 한다.

2. **조작은 ‘장난’이 아니다.**  
   합성 / 위조 / 날조는 실제 인간의 삶과 안전을 파괴한다. 우리는 그 피해자를 기본 값으로 둔다.

3. **검증 가능성은 누구에게나 열려야 한다.**  
   “이건 진짜인가?”를 확인하는 행위는 특정 기관의 특권이 아니라, 모두의 기본 툴이 되어야 한다.

이건 단순 윤리 선언이 아니라, 제품 요구사항입니다.  
우리는 ProofX와 LUDI Verifier가 이 기준을 깨지 않도록 설계합니다.
